# CS-305-Software-Security
Portfolio work for CS 305 - Software Security

## Artifact Included
This repository includes my completed Artemis Financial Practices for Secure Software Report. This artifact demonstrates my ability to identify software vulnerabilities, apply secure coding practices, and document security improvements.

## Reflection

### Client Summary
The client, Artemis Financial, is a financial services company that needed to improve the security of its software application. The company wanted to address vulnerabilities related to data protection, secure communication, and outdated dependencies. My role was to identify these issues and apply security improvements to reduce risk.

### What I Did Well
I did well identifying vulnerabilities and understanding how they impact software security. I used tools like OWASP Dependency-Check to analyze risks and learned how secure coding helps protect data and systems. Secure coding is important because it prevents attacks, protects sensitive information, and builds trust with users. It adds value to a company by reducing financial risk and protecting its reputation.

### Challenges and Helpful Areas
One challenging part was understanding the vulnerability scan results and determining which issues were critical. This was helpful because it taught me how to analyze and prioritize security risks instead of treating all warnings the same.

### Increasing Security Layers
I increased security by working with HTTPS, certificates, encryption concepts, and dependency scanning. In the future, I would continue using OWASP tools, secure coding practices, and manual code reviews to assess vulnerabilities and apply the best solutions.

### Functionality and Security
I tested the application after making changes to ensure it still worked correctly. After refactoring, I reviewed vulnerability scans again to confirm that security improved and no new issues were introduced.

### Tools and Practices Used
- OWASP Dependency-Check  
- Secure coding practices  
- HTTPS/SSL concepts  
- Certificate generation  
- Code testing and refactoring  

These tools will help me in future work by allowing me to identify and fix security issues early.

### What I Would Show Employers
I would show this report to demonstrate my ability to identify vulnerabilities, improve software security, and explain my decisions clearly. This shows I understand how to protect applications and data.
